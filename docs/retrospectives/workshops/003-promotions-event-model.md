# Retrospective — Workshops 003: the Promotions event model (DCB coupon redemption)

## Outcome summary

Ran the Promotions event-modeling pass the `promotions-dcb-workshop` handoff commissioned — the first build-facing design step of the direction ADR 024 chose. The handoff's named first decision (Workshop 003 vs. a Workshop 001 amendment) was presented to Erik with previews and a recommendation; he chose the **standalone Workshop 003**, the Workshop 002 precedent (new concept = new workshop). Three further modeling forks — the questions ADR 024 explicitly left to this pass — were then resolved the same way before drafting: the **coupon entry point** (cart-review field, UI-held, advisory query; the code becomes an event only at `PlaceOrder`), the **definition birth** (`DefineCoupon` → `CouponDefined`, CritterMart's first configuration-as-events use, seed-realized), and the **cancellation semantics** (release-on-cancel via a compensating tagged `CouponRedemptionReleased`). All three recommendations were accepted as presented.

**Workshop 003 (v1.0)** is the deliverable: slices **6.1 define / 6.2 validate / 6.3 redeem-with-DCB / 6.4 release-on-cancel** (continuing the workshop-global numbering after Workshop 002's 5.x), three new Orders-store events (`CouponDefined`; tagged `CouponRedeemed` and `CouponRedemptionReleased`), the `CouponUsage` **DCB boundary state** explicitly fenced off from the advisory `CouponUsageView` projection, proportional W2/W3 wireframe deltas per ADR 016, and GWT scenarios including the two the handoff's Definition of Done made mandatory: the **cap-breach → placement-rejected** path and the **`DcbConcurrencyException` race** (both boundary reads see `count < N`; one commits, the loser retries into the breach rejection — exactly one racing order survives). DCB mechanics are deferred to the `marten-advanced-dynamic-consistency-boundary` skill and ADR 024's verified Marten 9.11 symbols, not re-derived. Workshop 001 was touched **minimally** per the chosen fork: v1.13 resolves long-road parking-lot item 7 by pointer; no slice, GWT, or table edits — slices 4.1/4.5–4.7 are unchanged when no coupon is present, with the coupon-carrying variants modeled as *compositions* in Workshop 003.

## What worked

- **Presenting the ADR's forward questions as pre-drafting forks, not authoring choices.** ADR 024 deliberately left the entry point, the definition home, and (implicitly) the compensation story to this pass. Resolving all three with Erik *before* writing a line of the workshop — the retro-005 methodology — meant the model was fully determined at drafting time; nothing had to be walked back mid-artifact, and each fork's preview doubled as a section draft (the W2 wireframe in the entry-point preview landed nearly verbatim in § 5.1).
- **The fork previews carried the load-bearing constraint, not just the options.** The entry-point previews led with *which streams get written*; the cancellation previews led with the `count++ / count--` view arithmetic. Erik picked all three recommendations — the constraint-first framing retro 005 encoded ("lead with the constraint that makes it a fork") held up for domain-shape forks, not just architecture ones.
- **The advisory-vs-authoritative split fell out of the entry-point fork and became the workshop's best teaching seam.** Choosing UI-held entry forced the question "what does the cart-review check actually promise?" — answer: nothing. That produced the `CouponUsageView` (advisory, may lag) vs. `CouponUsage` (boundary state, computed at write, never persisted) distinction in §§ 4/7, which is the cleanest one-paragraph explanation of *why DCB exists* the doc set now has.
- **Global slice numbering across workshops paid for itself.** Workshop 002 taking 5.x made 6.x unambiguous for Promotions with zero coordination cost, and lets Workshop 001's resolved pointer cite "slices 6.1–6.4" without qualification.
- **The configuration-as-events adjunct finally earned a use.** The event-modeling skill has named the Bruun pattern since round one with "no clear CritterMart use"; coupon definitions (cap *N* as an event-sourced fact with an audit trail, seed-issued now, service-published later) are its textbook entry — and completing the adjunct-pattern set (Klefter, temporal automation, now configuration-as-events) is itself talk material.

## What was harder / notable

- **The compensation fork had a hidden interaction with terminal-guard discipline that needed its own GWT.** Release-on-cancel is only sound if a redemption can never be released twice; that guarantee comes from Workshop 001's existing rule that `OrderCancelled` is terminal and appended once, not from anything coupon-specific. The "at most one release per redemption" edge scenario exists to pin that inheritance explicitly — without it, the net-count arithmetic looks like it needs its own idempotency machinery.
- **Keeping the Workshop 001 amendment honest-but-minimal took discipline.** `PlaceOrder` gaining an optional `couponCode` field technically changes slice 4.1's command shape. The chosen fork said minimal amendment, so the change is modeled as a *composition* in Workshop 003 (§ 4 "PlaceOrder — amended shape"; § 6.3's no-coupon edge asserting 4.1 unchanged) with Workshop 001's pointer naming that arrangement — rather than editing 4.1's GWTs. The per-slice OpenSpec proposal must respect that split: the `order-lifecycle` capability will carry the delta, but the *model* home is Workshop 003.
- **The retrospectives README index was found one behind again** (`workshops/ (1)` with two files in the folder — the 002 entry never landed), the same drift the 038 session found on the implementations count. Back-filled to 3 in this session's bump. Two consecutive discoveries suggest the count-paragraph format invites drift; noted below.
- **A new `*(dcb)*` slice-table marker was coined.** Workshop 001's marker vocabulary (`*(query)*`, `*(scheduled)*`, `*(system)*`, `*(saga)*`) had no way to say "this write runs inside a Dynamic Consistency Boundary," and the marker is exactly where downstream OpenSpec authoring will look. Coined in Workshop 003 § 5 with an inline definition; future workshops inherit it.

## Methodology refinements

- **When an ADR names forward questions "for the workshop," turn them into session-start AskUserQuestion forks verbatim.** The ADR already did the work of isolating what is genuinely open; the workshop session's job is to close them with the owner before drafting, not to re-discover them mid-artifact. (Applied retro-005's fork discipline one layer down the pipeline.)
- **Fence write-side boundary state from read models by name, in the vocabulary section.** `CouponUsage` (never persisted) vs. `CouponUsageView` (advisory projection) carrying deliberately similar names with an explicit NOT-a-view fence prevented every downstream ambiguity while drafting. Any future DCB modeling should name its boundary state in § 4 with the same fence.
- **Index-count paragraphs drift; verify the folder before bumping.** Two sessions running found `docs/retrospectives/README.md` behind its folder. Cheap rule: any session touching the count globs the folder first and back-fills silently, noting it in the retro.
