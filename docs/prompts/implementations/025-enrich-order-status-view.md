## Framing

W4 order-tracking (PR #64) shipped against `OrderStatusView = { id, customerId, status, lines, total }` and logged two honest gaps: the view carries **no placement timestamp** (the W4 wireframe's `Placed 2026-06-14 14:02 UTC` line was dropped) and **no cancellation reason** (`OrderCancelled` folds to a bare `Status = "cancelled"` at `OrderStatusView.cs:44`, so the three reasons known to Narrative 004 — `stock_unavailable` / `payment_declined` / `payment_timeout` — never reach the screen; W4 shows a generic "Cancelled"). The workshop § 5.1 v1.10 amendment fenced both to "a future *enrich `OrderStatusView`* slice." **This is that slice.**

The data is already on the streams — this is a pure read-model enrichment, not new modeling:

- **`cancelReason`** — `OrderCancelled` **already carries** its `Reason` (with the `CancelReason` constants class, `OrderCancelled.cs`). The view's `Apply(OrderCancelled …)` simply drops it today. Folding `e.Reason` is a one-line change.
- **`placedAt`** — the placement time *is* the genesis `OrderPlaced` event's append timestamp, recorded by Marten as event metadata. Surface it via the `IEvent<T>` wrapper exactly as `CartView` surfaces `LastActivityAt` (`CartView.cs:23`: `Create(IEvent<CartCreated> e) => new(…, LastActivityAt: e.Timestamp)`). The `OrderPlaced` payload is **not** touched.

This is the cleanest demonstration of the ADR 020 split shipped to date: the **read** view surfaces a placement timestamp the **write** aggregate deliberately never stored (`Order.cs:23` — "the order tracks no activity timestamp, so no `IEvent<T>` wrapper is needed"). The view diverges from the aggregate, which is exactly what ADR 020 said it was free to do.

## Goal

`OrderStatusView` carries `DateTimeOffset PlacedAt` (set at genesis from `IEvent<OrderPlaced>.Timestamp`) and `string? CancelReason` (null until cancellation, then the `OrderCancelled` reason), with the existing `{ id, customerId, status, lines, total }` preserved as a **superset prefix** (additive only — nothing renamed or removed), so W3's place-order read is untouched. W4 binds both: a real placed-at line and per-reason cancellation copy, replacing the hardcoded sketch / generic "Cancelled." Full backend suite green (`dotnet test`), `dotnet format` clean; `client/` build + vitest green; a live boot confirms W4 renders the real placed-at + the right per-reason copy across a place→confirm and a cancel route.
