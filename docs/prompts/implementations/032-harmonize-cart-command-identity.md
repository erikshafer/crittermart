## Framing

The cart's identity transport was split: the **read** (`GET /carts/mine`, slice 3.5) keys identity off the `X-Customer-Id` header — the ADR 009 seam, the stand-in for a Polecat claim — but the three **commands** keyed it off the route (`/carts/{customerId}/…`). Slices 3.1–3.3 logged this as a deliberate deferral and the frontend SKILL told call sites not to "fix" it *until* a backend+OpenSpec harmonization landed. This slice **is** that harmonization. Route-keyed identity also lets any client act as any customer by editing the path; the header is the seam that becomes the authenticated claim, so converging the commands onto it is both a consistency and an identity-hygiene win — and makes the Polecat promotion a localized header→claim swap for the whole cart surface, not just the read.

## Goal

`POST /carts/mine/items`, `POST /carts/mine/items/{sku}/quantity`, and `DELETE /carts/mine/items/{sku}` resolve the customer from the `X-Customer-Id` header (blank/missing → `400`, distinct from the `409` of `NoOpenCart`/`CartItemNotPresent`); every other cart behavior is byte-for-byte unchanged. The full backend suite is green (`dotnet test`) with a new missing-header → 400 test per command; `dotnet format` clean; `client/` vitest + tsc + vite build green with the mutation hooks header-keyed and the call-site assertions proving identity rides the header, never the path. The OpenSpec `shopping-cart` change validates `--strict`.

## Spec delta

The OpenSpec **`shopping-cart`** capability gains **3 MODIFIED requirements** — *Add an item to the cart*, *Remove an item from the cart*, *Change a cart item's quantity* — each gaining the explicit identity-transport rule already stated on *Read the Customer's open cart* (identity via `X-Customer-Id` header, not route/body; missing header → `400`) + a new "reject with no customer identity" scenario. **Narrative 005 → v1.9** records the transport change (a Moment 3 clause + a Document History row) though it is invisible to the screen journey. **Frontend SKILL Convention 4** flips its "known divergence … do not fix" block to "harmonized" (v8). Four-step closure: **this prompt names it → the session executes → the retro confirms → the OpenSpec change + Narrative 005 + the SKILL record it.**
